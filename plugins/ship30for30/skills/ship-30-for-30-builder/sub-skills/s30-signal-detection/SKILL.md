# Day 22: Make Noise, Listen For Signal

> **Trigger:** "Build Day 22" or "Signal Detection"

## What You'll Build

Identify which topics resonate with both you AND your readers by analyzing internal signals (enjoyment, ease, friction) against external signals (views, likes, comments, shares).

---

## Build Flow

1. **Read config** — Load `s30-config.md` for student's niche, topics, and Notion DB ID
2. **Read template** — Load `references/D22-content.md` for signal detection framework
3. **Customize** — Apply framework to student's existing content (past 22 days)
4. **Analyze** — Identify 3 topics you enjoyed + 3 topics readers engaged with most
5. **Deliver to Notion** (if connected):
   - Find Day 22 row in student's Notion database
   - Fill the **Template** column with signal analysis framework
   - Set **Status** to "In Progress"
6. **Fallback** (no Notion): Save to `workspace/day-22-signal-analysis.md`
7. **Confirm** — "Your signal analysis is ready. Review which topics resonated most, then write about 1 today."

---

## Review Criteria

When student shares their analysis, check:
- Identified 3 internal signals (enjoyment-based topics)
- Identified 3 external signals (engagement-based topics)
- Recognizes overlap where internal + external align (the true niche)
- Understands Lean Writing Framework (testing small before investing big)

---

## On Completion

Ask which topic they want to double-down on. Update Notion Status to "Shipped". Prepare for Day 23.
